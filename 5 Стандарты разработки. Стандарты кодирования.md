**МДК.02.01 Технология разработки программного обеспечения**

---

## **1. Введение в стандарты разработки**

### **1.1. Понятие и необходимость стандартов**
**Стандарты разработки** — это набор правил, соглашений и рекомендаций, обеспечивающих единообразие процесса создания программного обеспечения.

**Цели стандартизации:**
- Повышение читаемости кода
- Упрощение поддержки и развития
- Снижение количества ошибок
- Ускорение onboarding новых разработчиков

### **1.2. Уровни стандартов**
- **Международные** (ISO, IEEE)
- **Отраслевые** (специфичные для домена)
- **Корпоративные** (внутри компании)
- **Проектные** (конкретный проект)

---

## **2. Международные стандарты разработки**

### **2.1. ISO/IEC 12207**
Процессы жизненного цикла программного обеспечения:
- Основные процессы (заказ, поставка, разработка)
- Вспомогательные процессы (документирование, управление конфигурацией)
- Организационные процессы (управление проектом, улучшение процессов)

### **2.2. ISO/IEC 25010**
Качество программного обеспечения:
- Функциональная пригодность
- Производительность
- Совместимость
- Надежность
- Безопасность

### **2.3. IEEE 1016**
Структура описания архитектуры ПО (SAD):
- Многопредставленческий подход
- Стандартизация документации

## **3. Стандарты кодирования**

### **3.1. Общие принципы**

#### **Принципы чистого кода:**
- **KISS** (Keep It Simple, Stupid)
- **DRY** (Don't Repeat Yourself)
- **YAGNI** (You Ain't Gonna Need It)
- **SOLID** (принципы объектно-ориентированного дизайна)

#### **Качество кода:**
- Читаемость и понятность
- Единообразие стиля
- Самодокументируемость
- Отсутствие "запахов кода"

### **3.2. Специфические стандарты по языкам**

#### **Java:**
- **Google Java Style Guide**
- **Oracle Code Conventions**
- Правила именования (camelCase для переменных, PascalCase для классов)

**Пример:**
```java
// Хорошо
public class UserService {
    private String userName;
    
    public void updateUserProfile(UserProfile profile) {
        // логика
    }
}

// Плохо
public class user_service {
    private String UserName;
    
    public void Updateuserprofile(UserProfile PROFILE) {
        // логика
    }
}
```

#### **Python:**
- **PEP 8** — основной стандарт
- **PEP 257** — документация (docstrings)
- **Black** — автоматический форматировщик

**Пример:**
```python
# Хорошо
def calculate_average(numbers: list) -> float:
    """Calculate the average of numbers in list.
    
    Args:
        numbers: List of numbers
        
    Returns:
        float: Average value
    """
    return sum(numbers) / len(numbers)

# Плохо
def calc_avg(n):
    return sum(n)/len(n)
```

#### **C/C++:**
- **Google C++ Style Guide**
- **C++ Core Guidelines**
- **GNU Coding Standards**

**Пример:**
```cpp
// Хорошо
class UserAccount {
public:
    std::string GetUserName() const { return user_name_; }
    
private:
    std::string user_name_;
    int account_id_;
};

// Плохо
class userAccount {
public:
    std::string getUserName() const { return username; }
    
private:
    std::string username;
    int accountId;
};
```

#### **JavaScript:**
- **Airbnb JavaScript Style Guide**
- **Google JavaScript Style Guide**
- **Standard JS**

**Пример:**
```javascript
// Хорошо
const MAX_USERS = 100;

class UserService {
    constructor() {
        this.users = [];
    }
    
    addUser(user) {
        if (this.users.length < MAX_USERS) {
            this.users.push(user);
        }
    }
}

// Плохо
const max_users = 100;

class userService {
    constructor() {
        this.Users = [];
    }
    
    AddUser(User) {
        if (this.Users.length < max_users) {
            this.Users.push(User);
        }
    }
}
```

### **3.3. Правила именования**

#### **Общие соглашения:**
- **camelCase** — переменные, функции, методы
- **PascalCase** — классы, интерфейсы, пространства имен
- **UPPER_CASE** — константы
- **kebab-case** — файлы, URL

#### **Осмысленные имена:**
```java
// Хорошо
int elapsedTimeInDays;
List<Customer> activeCustomers;

// Плохо
int etd;
List<Customer> acList;
```

### **3.4. Структура кода и форматирование**

#### **Отступы и пробелы:**
- Единый размер отступа (2 или 4 пробела)
- Пробелы вокруг операторов
- Пустые строки для логического разделения

#### **Длина строки:**
- Обычно 80-120 символов
- Перенос длинных выражений

**Пример:**
```python
# Хорошо
result = (very_long_variable_name + 
          another_long_variable - 
          complex_calculation)

# Плохо
result = very_long_variable_name + another_long_variable - complex_calculation
```

## **4. Документирование кода**

### **4.1. Комментарии**
- Объяснение "почему", а не "что"
- Избегание избыточных комментариев
- Актуальность комментариев

### **4.2. Документирующие комментарии**
- **JavaDoc** (Java)
- **XML Documentation** (C#)
- **Docstrings** (Python)
- **JSDoc** (JavaScript)

**Пример JavaDoc:**
```java
/**
 * Calculates the area of a circle.
 *
 * @param radius the radius of the circle
 * @return the area of the circle
 * @throws IllegalArgumentException if radius is negative
 */
public double calculateCircleArea(double radius) {
    if (radius < 0) {
        throw new IllegalArgumentException("Radius cannot be negative");
    }
    return Math.PI * radius * radius;
}
```

## **7. Практическое задание**

### **Задача: Анализ и улучшение кода**

**Дано:**
```python
def calc(x,y):
    # calculate sum
    r=x+y
    return r

class data_processor:
    def __init__(self):
        self.d=[]
    
    def add_data(self, d):
        self.d.append(d)
    
    def process_all_data(self):
        t=0
        for i in self.d:
            t=t+i
        return t
```

**Требуется:**
1. Проанализировать нарушения стандартов
2. Переписать код согласно PEP 8
3. Добавить документацию
4. Улучшить именование

**Решение:**
```python
class DataProcessor:
    """Processor for numerical data calculations."""
    
    def __init__(self) -> None:
        self._data = []
    
    def add_data(self, value: float) -> None:
        """Add a value to the dataset.
        
        Args:
            value: Numerical value to add
        """
        self._data.append(value)
    
    def calculate_sum(self) -> float:
        """Calculate the sum of all data values.
        
        Returns:
            float: Sum of all values in dataset
        """
        return sum(self._data)


def calculate_sum(a: float, b: float) -> float:
    """Calculate the sum of two numbers.
    
    Args:
        a: First number
        b: Second number
        
    Returns:
        float: Sum of a and b
    """
    return a + b
```

## **9. Метрики качества кода**

### **9.1. Основные метрики:**
- **Сложность кода** (цикломатическая сложность)
- **Coverage** — покрытие тестами
- **Technical Debt** — технический долг
- **Code Duplication** — дублирование кода

### **9.2. Мониторинг качества:**
- Регулярный code review
- Статический анализ
- Метарики в CI/CD
- Ретроспективы качества

---

**Заключение:** Стандарты разработки и кодирования являются критически важным элементом профессиональной разработки программного обеспечения. Они обеспечивают предсказуемость, поддерживаемость и качество кода, что напрямую влияет на успех проекта. Внедрение и соблюдение стандартов должно быть не формальностью, а осознанным выбором команды, направленным на создание лучшего продукта.
