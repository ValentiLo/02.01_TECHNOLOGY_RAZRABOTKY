## **Этап 2: Разработка технического задания (КТЗ)**

### **Техническое задание на разработку приложения "ActiveLife Fitness"**

**1. Общие положения**
- Наименование проекта: "ActiveLife Fitness"
- Заказчик: Фитнес-центр "ActiveLife"
- Исполнитель: [Команда разработчиков]
- Срок выполнения: 3 месяца

**2. Цели и задачи**
**Основные цели:**
- Увеличение лояльности клиентов на 25%
- Снижение количества пропущенных тренировок на 30%
- Автоматизация процесса записи на тренировки

**Задачи:**
- Разработка системы онлайн-записи
- Создание личного кабинета пользователя
- Реализация системы уведомлений
- Разработка модуля статистики и отчетности

**3. Функциональные требования**

**3.1. Модуль авторизации:**
- Регистрация по email/телефону
- Восстановление пароля
- Привязка к карте клиента

**3.2. Модуль расписания:**
- Просмотр расписания групповых тренировок
- Online-запись/отмена записи
- История посещений

**3.3. Модуль уведомлений:**
- Push-уведомления за 2 часа до тренировки
- Напоминания об оплате абонемента
- Новости фитнес-центра

**4. Нефункциональные требования**
- Время отклика интерфейса: не более 2 секунд
- Поддержка iOS 12+ и Android 8+
- Одновременная работа 500+ пользователей
- 99% доступность в рабочее время

**5. Критерии приемки**
- [ ] Успешная регистрация 100 тестовых пользователей
- [ ] Обработка 50 одновременных записей на тренировки
- [ ] Рассылка 1000+ push-уведомлений без ошибок
- [ ] Positive feedback от 80% тестовой группы

## **Этап 3: Проектирование системы**

### **Архитектура системы:**

```
┌─────────────────┐    ┌──────────────────┐    ┌─────────────────┐
│   Мобильное     │    │    API Gateway   │    │   База данных   │
│   приложение    │◄──►│                  │◄──►│   PostgreSQL    │
│                 │    │                  │    │                 │
└─────────────────┘    └──────────────────┘    └─────────────────┘
                              │
                              ▼
┌─────────────────┐    ┌──────────────────┐
│   Админ-панель  │    │  Сервис рассылки │
│   (React)       │    │  уведомлений     │
└─────────────────┘    └──────────────────┘
```

### **Модель данных:**

```sql
-- Пользователи
Users (id, email, phone, name, birth_date, created_at)

-- Тренировки
Workouts (id, name, description, trainer_id, datetime, max_participants)

-- Записи на тренировки
Bookings (id, user_id, workout_id, status, created_at)

-- Абонементы
Subscriptions (id, user_id, type, start_date, end_date, status)
```

### **API endpoints:**

```javascript
// Авторизация
POST /api/auth/login
POST /api/auth/register

// Расписание
GET /api/workouts?date=2024-01-15
POST /api/workouts/{id}/book
DELETE /api/bookings/{id}

// Пользователь
GET /api/user/profile
GET /api/user/bookings
```

## **Этап 4: Бриф на дизайн**

### **Дизайн-бриф для мобильного приложения "ActiveLife Fitness"**

**1. Целевая аудитория**
- Возраст: 18-45 лет
- Уровень дохода: средний и выше
- Интересы: здоровый образ жизни, спорт, саморазвитие

**2. Ключевые сценарии использования**
- Быстрая запись на тренировку за 3 клика
- Просмотр своего расписания на неделю
- Отслеживание прогресса посещений

**3. Требования к UI/UX**
**Стиль:**
- Минимализм, чистые линии
- Яркие акцентные цвета (оранжевый, зеленый)
- Четкая типографика

**Навигация:**
- Bottom navigation с 4 основными разделами
- Интуитивная иконография
- Быстрый доступ к основным функциям

**4. Основные экраны**
**4.1. Онбординг:**
- 3 экрана с преимуществами приложения
- Простая регистрация

**4.2. Главный экран:**
- Расписание на сегодня
- Ближайшие тренировки
- Быстрый доступ к записи

**4.3. Расписание:**
- Недельный view
- Цветовая кодировка типов тренировок
- Индикатор доступных мест

**5. Технические требования к дизайну**
- Формат: Figma
- Grid system: 8pt
- Шрифты: SF Pro (iOS), Roboto (Android)
- Разрешения: @1x, @2x, @3x

**6. Сроки и этапы**
- 1 неделя: Wireframes и user flow
- 2 неделя: Visual design основных экранов
- 1 неделя: Адаптация и подготовка assets
